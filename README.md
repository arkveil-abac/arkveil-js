# Arkveil SDK

Arkveil is the first lightweight and comprehensive ABAC platform, designed to bring fine-grained security into your applications with clarity and control. It lets you express complex access logic through simple, structured formulas.

## 🚀 Quick Start

### NestJS

```bash
npm install @arkveil/nest
```

```typescript
import { Module } from "@nestjs/common";
import { ArkveilModule } from "@arkveil/nest";

@Module({
  imports: [
    ArkveilModule.forRoot({
      serviceUrl: "https://api.arkveil.com",
      apiKey: "your-api-key",
      getUserAttributes: (req) => ({ id: req.user?.id }),
    }),
  ],
})
export class AppModule {}
```

Use the `@PermissionPoint` decorator:

```typescript
import { Controller, Get } from "@nestjs/common";
import { PermissionPoint } from "@arkveil/nest";

@Controller("articles")
export class ArticlesController {
  @Get("/admin")
  @PermissionPoint("content-service.article-delete")
  adminAction() {
    return "Protected content";
  }
}
```

### Express / Node.js

```bash
npm install @arkveil/node arkveil
```

```typescript
import { Arkveil } from "@arkveil/node";

const arkveil = new Arkveil({
  serviceUrl: "https://api.arkveil.com",
  apiKey: "your-api-key",
  getUserAttributes: (req) => ({ id: req.user?.id }),
});

app.post(
  "/api/admin",
  arkveil.permissionPoint("content-service.article-delete"),
  (req, res) => {
    res.json({ message: "Protected content" });
  }
);
```

### Core SDK (Runtime Agnostic)

```bash
npm install arkveil
```

```typescript
import { Arkveil } from "arkveil";

const arkveil = new Arkveil({
  serviceUrl: "https://api.arkveil.com",
  apiKey: "your-api-key",
});

const result = await arkveil.checkPermission({
  code: "content-service.article-delete",
  user: { id: "user-123" },
  context: {},
});

if (result.granted) {
  // Allow access
} else {
  // Deny access
}
```

## 🧬 Typed Codes & Attributes

The Arkveil CLI generates one TypeScript file that types the SDK against your
project — permission codes **and** `user`/`context` attributes. Wire it in once
and every place that takes a `code` (`checkPermission`, the Node
`permissionPoint` middleware, the NestJS `@PermissionPoint` decorator) plus the
`getUserAttributes` / `getContextAttributes` extractors get autocomplete and
reject unknown or mistyped values at compile time.

### 1. Generate the file with the CLI

```bash
# Reads your codes + user/context JSON Schemas and writes the typed file.
arkveil generate typescript -o src/arkveil.generated.ts

# See the full recipe (and what each registry maps to):
arkveil sdk info
```

The generated `arkveil.generated.ts` looks like this:

```typescript
// AUTO-GENERATED by `arkveil generate typescript`. Do not edit by hand.
export type ArkveilCodes =
  | "content-service.article-delete"
  | "user-service.user-create";

export interface ArkveilUserAttributes {
  id?: string;
  role: "admin" | "editor" | "viewer";
}

export interface ArkveilContextAttributes {
  ipAddress?: string;
  region?: "EU" | "US";
}

// Register globally so the default `Arkveil` generics, the `permissionPoint`
// middleware, and the `@PermissionPoint` decorator pick everything up.
declare module "arkveil" {
  interface ArkveilCodeRegistry {
    codes: ArkveilCodes;
  }
  interface ArkveilUserRegistry {
    attributes: ArkveilUserAttributes;
  }
  interface ArkveilContextRegistry {
    attributes: ArkveilContextAttributes;
  }
}
```

### 2. Everything is typed automatically

```typescript
import { Arkveil } from "@arkveil/node";
import "./arkveil.generated"; // side-effect import applies the augmentation

const arkveil = new Arkveil({
  serviceUrl,
  apiKey,
  getUserAttributes: (req) => ({ role: req.user.role }), // ✅ checked vs registry
  // getUserAttributes: () => ({ role: "superuser" }),   // ❌ compile error
});

arkveil.permissionPoint("content-service.article-delete"); // ✅ autocompletes
arkveil.permissionPoint("typo"); // ❌ compile error

await arkveil.checkPermission({
  code: "content-service.article-delete",
  user: { role: "admin" }, // ✅ autocompletes, rejects unknown keys
  context: { region: "EU" },
});
```

The NestJS `@PermissionPoint` decorator is typed by the same augmentation — no
extra wiring:

```typescript
@PermissionPoint("content-service.article-delete") // ✅ autocompletes
```

### Prefer explicit generics over global augmentation?

Pass the generated types directly instead of importing the file for its
side effect:

```typescript
import type {
  ArkveilCodes,
  ArkveilUserAttributes,
  ArkveilContextAttributes,
} from "./arkveil.generated";

const arkveil = new Arkveil<
  ArkveilCodes,
  ArkveilUserAttributes,
  ArkveilContextAttributes
>({ serviceUrl, apiKey });

// NestJS: build a typed decorator once and re-export it.
import { createPermissionPoint } from "@arkveil/nest";
export const PermissionPoint = createPermissionPoint<ArkveilCodes>();
```

Until the registry is augmented, codes stay `string` and `user` / `context` stay
`Record<string, any>`, so untyped usage keeps working.

## ✨ Features

### NestJS SDK (`@arkveil/nest`)

- 🔒 **Declarative Permission Checks** - Use `@PermissionPoint` decorator
- 🌐 **Global Module** - Configure once, use everywhere
- 🔄 **Async Configuration** - Support for ConfigService and dependency injection
- 📡 **Multi-Protocol Support** - HTTP, GraphQL, and WebSocket contexts
- 🎯 **Type-Safe** - Full TypeScript support

### Node.js SDK (`@arkveil/node`)

- 🚀 **Framework Agnostic** - Works with Express, Fastify, and more
- 🔌 **Middleware Pattern** - Easy integration with existing apps
- 🎨 **Customizable** - Custom user extraction and denial handlers
- ⚡ **Fast** - Optimized for performance

### Core SDK (`arkveil`)

- 🌍 **Runtime Agnostic** - Works in any JavaScript environment
- 🔧 **Flexible** - Build your own platform-specific implementations
- 📦 **Lightweight** - Minimal dependencies
- 🔄 **Retry Logic** - Built-in retry mechanism for failed requests
- 🧬 **Typed Codes & Attributes** - Permission codes and `user`/`context`
  attributes typed from your project's schemas (see above)
